## 定位权限说明

### `NSLocationWhenInUseUsageDescription`

**使用期间定位权限**。App 在前台运行时才能获取位置，切到后台就停止定位。

```xml
<!-- Info.plist 中配置 -->
<key>NSLocationWhenInUseUsageDescription</key>
<string>需要获取您的位置来显示附近的新闻</string>
```

### `NSLocationAlwaysUsageDescription`

**始终定位权限**。App 在前台和后台都能获取位置。

```xml
<key>NSLocationAlwaysUsageDescription</key>
<string>需要持续获取您的位置来推送附近的信息</string>
```

### 对比

| | WhenInUse | Always |
|---|---|---|
| 前台定位 | ✅ | ✅ |
| 后台定位 | ❌ | ✅ |
| 用户感知 | 状态栏无提示 | 状态栏显示蓝色定位图标 |
| 隐私级别 | 低 | 高 |
| 审核难度 | 容易通过 | 需要充分理由 |

## iOS 9 关于权限的新特性

### 1. 强制要求配置权限描述字符串

iOS 8 开始要求，iOS 9 **强制执行**。如果不在 `Info.plist` 里写描述字符串，调用定位直接**静默失败**，不弹授权弹窗。

```objc
// 代码请求权限
[locationManager requestWhenInUseAuthorization];
// 如果 Info.plist 没配 NSLocationWhenInUseUsageDescription → 什么都不会发生
```

弹窗效果：
```
┌─────────────────────────────┐
│   "XX App"想要访问您的位置     │
│                             │
│  需要获取您的位置来显示附近的新闻  │  ← 你配的描述字符串
│                             │
│   [ 不允许 ]    [ 允许 ]      │
└─────────────────────────────┘
```

### 2. App Transport Security (ATS)

iOS 9 默认**禁止 HTTP 请求**，只允许 HTTPS。需要在 `Info.plist` 中配置例外：

```xml
<key>NSAppTransportSecurity</key>
<dict>
    <!-- 允许所有 HTTP 请求（开发时用） -->
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>
```

### 3. 相机/相册/通讯录等权限也需要描述

iOS 9 之后逐步对所有敏感权限都要求写描述字符串：

| Key | 权限 |
|-----|------|
| `NSLocationWhenInUseUsageDescription` | 使用时定位 |
| `NSLocationAlwaysUsageDescription` | 始终定位 |
| `NSCameraUsageDescription` | 相机 |
| `NSPhotoLibraryUsageDescription` | 相册 |
| `NSContactsUsageDescription` | 通讯录 |
| `NSMicrophoneUsageDescription` | 麦克风 |
| `NSCalendarsUsageDescription` | 日历 |

### 4. URL Scheme 白名单

iOS 9 要求在 `Info.plist` 中声明你要跳转的 App 的 URL Scheme：

```xml
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>weixin</string>    <!-- 微信 -->
    <string>alipay</string>    <!-- 支付宝 -->
</array>
```

不声明的话，`canOpenURL:` 直接返回 NO。

### 5. Bitcode 支持

iOS 9 新增 Bitcode（中间码），Xcode 默认开启。如果第三方库不支持 Bitcode，需要手动关闭：

```
Build Settings → Enable Bitcode → NO
```

## 总结

iOS 9 的核心主题就是**加强隐私和安全**：

| 特性 | 一句话总结 |
|------|----------|
| 权限描述字符串 | 必须告诉用户你为什么要这个权限 |
| ATS | 默认禁止 HTTP，强推 HTTPS |
| URL Scheme 白名单 | 不能随意探测用户装了哪些 App |
| Bitcode | 让苹果优化你的二进制包 |
