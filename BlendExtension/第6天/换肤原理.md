## 换肤原理分析

### 一、资源文件结构

```
skin/
├── red/
│   ├── face.png
│   ├── heart.png
│   ├── rect.png
│   └── SkinColors.plist     ← 颜色配置
├── green/
│   ├── face.png
│   ├── heart.png
│   ├── rect.png
│   └── SkinColors.plist
├── blue/    ...
└── orange/  ...
```

每套皮肤：**同名图片 + 颜色配置**。换肤 = 换文件夹。

---

### 二、核心工具类 HMSkinTools

整个换肤的核心就是这个工具类，负责三件事：

#### 1. 初始化（`+initialize`）

```objc
+ (void)initialize {
    // 从偏好设置读取上次保存的皮肤名，默认 green
    _skinName = [[NSUserDefaults standardUserDefaults] objectForKey:@"skinName"];
    if (_skinName == nil) _skinName = @"green";
    
    // 加载颜色配置
    [self loadColorDict];
}
```

`+initialize` 是类第一次被使用时自动调用，保证只初始化一次。

#### 2. 获取图片（动态拼接路径）

```objc
+ (UIImage *)imageWithImageName:(NSString *)imageName {
    // 拼接路径：skin/当前皮肤名/图片名
    NSString *path = [NSString stringWithFormat:@"skin/%@/%@", _skinName, imageName];
    return [UIImage imageNamed:path];
}
```

**换肤 = 换文件夹路径**

```
skin/green/face.png  →  绿色皮肤的脸
skin/red/face.png    →  红色皮肤的脸
```

文件名一样，只是换了文件夹，图片内容不同。

#### 3. 获取颜色（从 plist 读取）

```objc
+ (void)loadColorDict {
    // 读取当前皮肤的 SkinColors.plist
    // "0,255,0,1" → UIColor(r:0, g:255, b:0, a:1)
    // 转换后存入 _colorDict 缓存
}
```

```
SkinColors.plist（green）          SkinColors.plist（red）
labelTextDayColor = "0,255,0,1"    labelTextDayColor = "255,0,0,1"
        ↓                                   ↓
   绿色文字                             红色文字
```

#### 4. 保存皮肤（持久化）

```objc
+ (void)saveSkinName:(NSString *)skinName {
    _skinName = skinName;           // 更新内存中的皮肤名
    [self loadColorDict];           // 重新加载颜色配置
    [[NSUserDefaults standardUserDefaults] setObject:skinName forKey:@"skinName"];
    // App 下次启动仍然是这套皮肤
}
```

---

### 三、完整换肤流程

```
用户点击"红色"按钮
    ↓
[HMSkinTools saveSkinName:@"red"]
    ↓
_skinName = @"red"
重新加载 skin/red/SkinColors.plist 的颜色
保存到 NSUserDefaults
    ↓
[HMSkinTools imageWithImageName:@"face"]
    ↓
路径 = skin/red/face.png  ← 换了文件夹
    ↓
imageView.image = 红色皮肤的图片
label.textColor = 红色皮肤的颜色
label.backgroundColor = 红色皮肤的背景色
```

---

### 四、设计亮点

| 设计 | 作用 |
|------|------|
| `static _skinName` | 全局唯一皮肤名，所有页面共享 |
| `static _colorDict` | 颜色缓存，避免重复解析 plist |
| `NSUserDefaults` | 持久化皮肤选择，下次启动记住 |
| `+initialize` | 类第一次使用时自动初始化，保证只执行一次 |
| `extern NSString * const` | 颜色 key 用常量定义，防止拼写错误 |
| 同名图片不同文件夹 | 换肤只需换路径前缀，扩展新皮肤无需改代码 |

---

### 五、扩展新皮肤有多简单

只需要：

```
1. 新建 skin/purple/ 文件夹
2. 放入 face.png、heart.png、rect.png
3. 新建 SkinColors.plist 配置颜色
4. 加一个"紫色"按钮，调用 [HMSkinTools saveSkinName:@"purple"]
```

**不需要改任何逻辑代码**，这就是这套换肤方案的核心价值。
