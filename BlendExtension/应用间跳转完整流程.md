## iOS 应用间跳转完整流程

---

### 一、核心概念：URL Scheme

每个 App 可以注册一个**自定义协议头**（URL Scheme），就像网址有 `http://`，App 也可以有自己的协议：

```
http://www.baidu.com     → 打开浏览器
weixin://                → 打开微信
alipay://                → 打开支付宝
weixin5://               → 打开你的"微信"练习项目
```

---

### 二、配置步骤

#### 被跳转方（微信）— 注册 URL Scheme

一般在TARGETS中的info的URL types 中的URL Schemes中添加：
会自动在 `Info.plist` 中注册：

```xml
<key>CFBundleURLTypes</key>
<array>
    <dict>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>weixin5</string>   ← 注册协议头
        </array>
    </dict>
</array>
```

> 相当于告诉系统："我是 `weixin5` 协议的处理者"

#### 跳转发起方（网易新闻）— 添加白名单（iOS 9+）

在 `Info.plist` 中声明你要跳转的 Scheme：

```xml
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>weixin5</string>   ← 白名单
</array>
```

> 不加白名单，`canOpenURL:` 直接返回 NO

### 四、代码流程

#### 第 ① 步：发起方 — 检查 + 跳转

```objc
NSURL *url = [NSURL URLWithString:@"weixin5://timeline.news"];

// 先检查：对方 App 是否安装了
if ([[UIApplication sharedApplication] canOpenURL:url]) {
    // 跳转
    [[UIApplication sharedApplication] openURL:url 
                                       options:@{} 
                             completionHandler:nil];
}
```

#### 第 ② 步：系统处理

```
系统收到 openURL 请求
    ↓
查找哪个 App 注册了 "weixin5" 协议
    ↓
找到 → 打开微信 App
找不到 → 什么都不做
```

#### 第 ③ 步：被跳转方 — 接收回调

```objc
// 微信 AppDelegate.m
- (BOOL)application:(UIApplication *)app
            openURL:(NSURL *)url
            options:(NSDictionary *)options {
    
    NSString *URLStr = url.absoluteString;
    // URLStr = "weixin5://timeline.news"
    
    if ([URLStr containsString:@"timeline"]) {
        // 跳转到朋友圈页面
    } else if ([URLStr containsString:@"session"]) {
        // 跳转到聊天页面
    }
    
    return YES;
}
```

### 五、URL 的设计

URL 可以携带信息，告诉对方 App 该做什么：

```
weixin5://                    → 只是打开微信
weixin5://timeline            → 打开微信并跳到朋友圈
weixin5://session             → 打开微信并跳到聊天
weixin5://pay?amount=100      → 打开微信并发起支付（带参数）
```

---

### 六、完整配置清单

| 步骤 | 谁来做 | 做什么 |
|------|--------|--------|
| 1 | 被跳转方 | Info.plist 注册 `CFBundleURLSchemes` |
| 2 | 发起方 | Info.plist 添加 `LSApplicationQueriesSchemes` 白名单 |
| 3 | 发起方 | 代码调用 `canOpenURL:` 检查 |
| 4 | 发起方 | 代码调用 `openURL:options:completionHandler:` 跳转 |
| 5 | 被跳转方 | AppDelegate 实现 `application:openURL:options:` 接收 |
| 6 | 被跳转方 | 解析 URL，跳转到对应页面 |

---

### 七、注意事项

| 注意点 | 说明 |
|--------|------|
| iOS 9+ | 必须配白名单 `LSApplicationQueriesSchemes`，否则 `canOpenURL:` 返回 NO |
| iOS 10+ | 必须用 `openURL:options:completionHandler:`，旧方法被强制禁用 |
| Scheme 唯一性 | 自定义 Scheme 不要和其他 App 冲突，建议用反向域名如 `com.company.app` |
| 两个 App 都要安装 | 模拟器上两个项目都要先运行一次，安装到同一个模拟器 |
