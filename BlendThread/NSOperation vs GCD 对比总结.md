# 📊 NSOperation vs GCD 对比总结

| 特性 | GCD（Grand Central Dispatch） | NSOperation |
|------|-------------------------------|-------------|
| **推出时间** | iOS 4.0 | iOS 2.0（iOS 4.0 后重写） |
| **语言** | C 语言 | Objective-C（面向对象） |
| **核心单位** | 任务（Block） | 操作（NSOperation 对象） |
| **队列类型** | 串行、并行、全局、主队列 | 并发队列（NSOperationQueue） |
| **执行方式** | 同步 / 异步 | 异步（通常） |
| **依赖关系** | 需手动通过同步或调度组实现 | 内置 API，支持设置依赖 |
| **队列控制** | 基础控制 | 支持**暂停/继续**、**取消所有操作** |
| **并发控制** | 无直接API（需信号量等） | 支持设置**最大并发数** |
| **特有功能** | 一次性执行、延迟执行、调度组 | 依赖管理、任务取消、队列挂起 |
| **适用场景** | 轻量级并发、一次性任务、延迟执行 | 复杂任务依赖、可管理队列、需取消的任务 |

---

### 🧠 如何选择？
- **使用 GCD**：适合简单的并发任务、一次性/延迟执行、调度组场景。
- **使用 NSOperation**：适合需要**依赖关系**、**任务取消**、**队列控制**或**复杂任务管理**的场景。

### 📌 补充说明：
- **NSOperation** 底层也是基于 **GCD** 实现的，但提供了更高层次的抽象和更丰富的控制功能。
- 如果你已经在使用 **Swift Concurrency（async/await）**，在 iOS 13+/macOS 10.15+ 上可以考虑使用它作为现代替代方案，但 GCD 和 NSOperation 仍然是稳定可靠的选择。
